反射

2023-07-22
第四阶段
Word count: 1.1k   |   Reading time≈ 4 min

可以通过动态的方式完成一个类的操作，称之为是反射的方式。

正常方式：
new关键字实例化一个对象，通过对象调用类中的方法，属性
反射方式：
还是针对一个类操作，实例化，调用类中的方法，属性。

如果需要学习反射的使用，需要从java.lang.Class类开始，Class类是反射的源头,程序中任何一个类的执行，JVM都会自动创建一个Class对象，对象中记录运行的类中的所有的内容，

正常方式操作的是运行类的本身；
反射方式操作的是Class对象中的内容。

认识并使用Class，获取一个Class对象

通过Class.forName("类的全限定名")

类的全限定名即包名+类名，该方式只能针对类进行操作

反射的应用

实例化目标对象

默认调用目标类中的公共的无参数的构造方法进行实例化

可以打印实例化的对象或打印实例化对象的哈希码

从上述例子可以看出，两次获取的实例化对象哈希码不同，说明两次创建了不同的对象

通过反射实例化的对象，调用该对象的属性和方法不再是通过反射的方式

可使用目标类中的任意一个构造方法实例化对象

方法 说明
Constructor getConstructor(Class<?>… parameterTypes) 返回一个 Constructor 对象，它反映此 Class 对象所表示的类的指定公共构造方法（可以使含参构造）。
Constructor<?>[] getConstructors() 返回一个包含某些 Constructor 对象的数组，这些对象反映此 Class 对象所表示的类的所有公共构造方法。
Constructor getDeclaredConstructor(Class<?>… parameterTypes) 返回一个 Constructor 对象，该对象反映此 Class 对象所表示的类或接口的指定构造方法（可以使含参构造）。
Constructor<?>[] getDeclaredConstructors() 返回 Constructor 对象的一个数组，这些对象反映此 Class 对象表示的类声明的所有构造方法。
